## 什么是真正的叙事？

**叙事不是信息包装，是发明视角。**

### 例子：用"编程"理解"文明"

**普通叙事**：日本经济停滞，因为人口老龄化和创新不足。

**深层叙事**：日本是"固化代码系统"——早期写死"精细、稳定、守规矩"的核心算法，工业时代跑分无敌，但互联网时代不能热更新，底层架构锁死。成也固定代码，败也固定代码。

**第一性原理**：死板代码困死文明，开放接口永生，动态适配称王。

## 核心功能

### 1. 叙事发明方法论
- 跨学科映射：用A领域的概念理解B领域
- 连续追问：5个"为什么"挖到底层
- 反向推导：从结果找关键变量
- 抽象提炼：从多个案例提取规律

### 2. 第一性原理库
- 代码决定命运
- 旧成功是新诅咒
- 分工因智能不足而存在
- 网络效应决定赢家
- 边际成本趋零改变一切
- 熵增定律

### 3. 跨学科思维映射库
- 编程 → 文明/公司/个人
- 进化论 → 商业
- 物理 → 组织
- 生物 → 经济
- 算法 → 人生
- 医学 → 社会

### 4. 经典叙事案例库
- Level 3案例：用编程理解文明
- Level 3案例：旧成功是新诅咒
- Level 2案例：分工因智能不足而存在
- Level 2案例：网络效应决定赢家
- 每个案例都有完整的推导过程和预测能力

### 5. 叙事能力检验标准
- 一句话说清楚
- 跨领域通用
- 能解释过去
- 能预测未来

## 使用方法

### 方法一：学习叙事思维
- 阅读本文件 + references/目录
- 用案例库训练叙事能力
- 从Level 1提升到Level 3

### 方法二：AI辅助叙事
- 将本文件作为prompt加载
- AI按方法论搜集信息、提炼叙事
- 人类负责第一性原理推导

### 方法三：持续迭代叙事
- 每天搜集新事件
- 用新事件验证/挑战现有叙事
- 建立自己的第一性原理库

---

## 如何检验叙事质量？

### 好叙事的四个特征

1. **一句话能说清**
   - "死板代码困死文明" ✅
   - "日本经济停滞是因为多个因素" ❌

2. **跨领域通用**
   - "代码决定命运"适用于文明、公司、个人 ✅
   - "日本经济停滞"只适用于日本 ❌

3. **能解释过去**
   - 用"旧成功是新诅咒"解释诺基亚、柯达、日本 ✅

4. **能预测未来**
   - 预测：当前成功的公司会死于下一个时代 ✅
