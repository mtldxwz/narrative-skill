# 叙事 1.0

从纷繁信息中提炼有规律可循的叙事，帮你看到表象背后的本质。

**版本**：1.0
**更新日期**：2026年4月5日

---

## 为什么需要这个skill？

信息爆炸时代，稀缺的不是信息，是"框架"。

大多数人每天看到各种新闻、动态、热点——但他们只看到了"发生了什么"，没有看到"这背后意味着什么"。

叙事者不是讲故事，是**定义问题**。问题的价值往往大于答案。

---

## 这个skill做什么？

从多领域信息中提炼出有解释力的叙事，每条叙事包含：

1. **叙事本身** - 一句话概括一个趋势或规律
2. **信息来源** - 不止一个信源支撑
3. **学术概念** - 用学术理论解释现象
4. **我的看法** - 有人味的观点，不是冷冰冰的分析

---

## 核心叙事框架

### 千年叙事：分工的消失

人类社会因智能不足而分工。AI时代，当智力像自来水一样廉价，分工可能终结。

**支撑现象**：
- 程序员skill化（一个skill取代一个岗位）
- 医生、律师、会计师的专业壁垒被AI侵蚀
- "全栈"从技术概念变成社会概念

### 5年叙事：推理时代

训练时代已过，现在是推理时代。AI从"学习知识"转向"应用知识"。

**支撑现象**：
- 推理成本暴跌99%
- 边缘计算、端侧AI爆发
- AI芯片从训练导向转向推理导向

## 执行流程

### 步骤1：信息搜集
```
搜索关键词：
- AI类："AI news today"、"人工智能 最新"、"大模型 动态"
- 经济类："全球经济"、"利率"、"供应链"
- 社会类："社会新闻"、"人口"、"就业"

时间范围：最近24-48小时
```

### 步骤2：筛选事件
**筛选标准**：
- 有"叙事价值"（不是所有新闻都值得提炼）
- 能匹配核心叙事框架
- 有多个独立信息源

**过滤掉**：
- 纯产品发布（除非有深层意义）
- 营销软文
- 孤立事件（无法形成叙事）

### 步骤3：匹配框架
```
对每个事件问：
1. 这件事在千年叙事、五年叙事、还是当下叙事层级？
2. 它支撑/挑战/衍生哪个叙事？
3. 可以用哪个学术概念解释？
4. 和其他叙事有什么交叉？
```

### 步骤4：生成叙事
每条叙事包含：
- 【叙事】一句话概括（不超过30字）
- 【来源】2-3个信息来源
- 【概念】学术理论支撑
- 【我怎么看】个人观点（有人味，不冷冰冰）

### 步骤5：质量检查
**叙事质量评分**：
- 解释力：能否让人"看懂更多事情"？（1-5分）
- 人味：有没有"我怎么看"的温度？（1-5分）
- 证据强度：有多少独立信源支撑？（1-5分）
- 概念运用：学术概念是否自然融入？（1-5分）

**及格线**：总分≥12分

---

## 输出格式

每天生成10条叙事，分为三个维度：

**AI叙事（5条）**
- AI技术、应用、监管、伦理

**经济叙事（2条）**
- 宏观经济、产业变迁、全球化

**社会叙事（3条）**
- 人口结构、文化心理、社会问题

## 叙事者的本质

**叙事者不是讲故事，是定义问题。**

大多数人回答别人提出的问题：这个产品好不好？这个技术厉害不厉害？

叙事者提出问题本身。

**叙事者在塑造认知框架。**

当叙事被接受，人们不只是"知道了你的观点"，而是开始用你的框架看世界。

**每个时代都有它的核心叙事。**

- 工业时代：机器取代人力
- 信息时代：数据是新的石油
- AI时代：？——这个空白，就是机会
